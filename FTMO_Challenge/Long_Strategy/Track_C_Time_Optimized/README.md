# Track C - Time Optimized (MC >= 95%)

This track is designed to optimize how fast challenge phases are passed while preserving robustness.

## Goal
- Minimize average time to pass Step 2.
- Enforce Monte Carlo pass probability constraint: >= 95%.

## Method
- Evaluate fixed parameter sets on walk-forward out-of-sample windows.
- Run Monte Carlo on OOS trades for each candidate.
- Keep only candidates with MC pass probability >= 95%.
- Rank feasible candidates by:
  1. Lowest average trades to Step 2 (+15%),
  2. Lowest average trades to Step 1 (+10%),
  3. Highest MC pass probability,
  4. Highest OOS return.

These are the canonical outputs of the Track C time-to-pass optimizer.

Separate extended-validation artifacts exist in the same folder:
- `reports/wfv_extended_fold_results.csv`
- `reports/wfv_extended_summary.json`
- `reports/wfv_extended_oos_trades.csv`

Those `wfv_extended_*` files are generated by `scripts/walk_forward_extended_1y_2y.py`, which uses a different walk-forward design, parameter grid, and compliance-first scoring. They are not a direct validation of the single candidate saved in `track_c_best_candidate.*`.

## Run
From repo root:

```bash
python FTMO_Challenge/Long_Strategy/Track_C_Time_Optimized/scripts/optimize_time_to_pass.py
```

## Notes
- Step targets in this implementation are cumulative from initial capital:
  - Step 1: +10%
  - Step 2: +15%
- If you want strict phase-reset semantics for Step 2, adapt the timing logic to use a second phase after Step 1 completion.
- The effective Monte Carlo constraint for a checked-in result is whatever is recorded inside `reports/track_c_best_candidate.txt` or `reports/track_c_best_candidate.json`. The script default is 95%, but a run can override it via `--mc-constraint-pct`.
